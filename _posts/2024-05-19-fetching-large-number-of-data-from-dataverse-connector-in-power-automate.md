<img border="1" src="{{ site.baseurl }}/assets/images/035/img_25d9f54c30.png" style="display: none;" /><span>&nbsp;<span>&nbsp;&nbsp; &nbsp;</span></span>Handling a large amount of data remains a challenge for many Power Automate enthusiasts. Here's an article detailing how to retrieve a large number of records from Dataverse using Power Automate. There are three methods for fetching large number of data from Dataverse via Power Automate.<p></p><ul><li><a href="#threshold-way">Threshold Way</a></li><li><a href="#paging-cookie-way">Paging Cookie way</a></li><li><a href="#next-link-way">Next Link Way</a></li></ul><h2 id="threshold-way" style="text-align: left;">Threshold Way</h2><p>To demonstrate this, I've set up a cloud instant flow. In the flow editor page, beneath the trigger, you need to add an action called "List Rows from Dataverse" (you can rename it if you prefer).</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEgrkN97i02QheqTfwDgRbXO90fE55GWUvqPXzSfZlMocSzxYfrKn2u2-G4KPl-FWm0iQTFUbNBVD-cgdHODVprwjBxrrObmAi2kq4leIIqCJrj4cgEELw9CS9VdnQKKwpTFBA2mjmkbTgl35AT2n2rD316YV5CgqrmluwKtBZpS9-aoy4N2x00DYqC9xE4" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_6dfb2bd66e.png" /></a></div><br /><p></p><p>In the "List Rows" action, select the table name and add any necessary queries. For my purpose, I've chosen the Contacts table and selected only the fullname column from that table. Alternatively, Fetch XML can also be used for querying the same data.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjJU0rNH7EloUPPgxkmzEPzo-9GgYQ5LpFoS5ja5h5ZgBEOX-hdeAWlkBfkgU7IueaHtP8uteuZBvKKm2lzWVQ6zyfDBmnWVWzh6n4bnJ8mTkNjtYRAWL3XMg3sbo0X7ZWRMyoD1FEqYYHcMm2KHpNzPZ2OIpIdXbQ_a2ugsTpEfxqFyusOAPD9ZJKekdo" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_c6cefaaba7.png" /></a></div><p></p><p>The "List Rows" action of the Dataverse connector for Power Automate defaults to fetching 5000 records. However, by enabling Pagination in ellispse → Settings → Threshold (accepting values less than or equal to 100000), you can configure it to fetch records up to a count of 100000.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjcr9pissK3-Bn48ScPAiEF9u7uPxCYtTs2zLk_UuVV06-T-9N8t9YP_YZKzmuY8NOCutNocTsQV4fsjGHA36ZKK-Ukg39of6MTZMzqKOR0Boo9ZBbK_C9zVkSiUmAD9CrvqcHYfdN4JWbzzzXIiTSlsPF24n1ZVNKlHAg2KhFB032Ikb6G8ioQci2xBKQ" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_5638ae2be7.png" /></a></div><p></p><p>Add another compose action to view the records count. For that, write an expression to find the length of the outputs from the "List Contacts" action. Here's the expression provided:</p><pre><code class="language-jsx">length(outputs('List_Contacts')?['body/value'])</code></pre><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEiz14U3pl5f3gHRSEKFqGu1PtYxppdNCfo61NYhZngUDMSrq1OQ8IUc9jQmpryzfU_FQ7mmbYJ0fwSvA3pZIRQjwzk_uLp5kjh4L1pTig0vk0gktK5f47cw65e6ID0L0SS_hvzmmtViC2rSHbuUKaPGGBL8fYYzGS08JfVSQchFrUhxV1jxT3DRYivY8l8" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_bfaf6c3cf8.png" /></a></div><p></p><p style="text-align: justify;">Save the flow and run it. Wait until the flow runs successfully. Open the run and expand the compose action; there you will see the record count. In my case, I have more than 100k records. However, since we set the threshold limit to 100,000 in the standard configuration, it will fetch records up to that limit.</p><h2 id="paging-cookie-way" style="text-align: justify;"><span style="text-align: left;">Paging Cookie way</span></h2><p style="text-align: justify;"><span>&nbsp;&nbsp; &nbsp;</span>The Dataverse connector for Power Automate has a threshold limit of 100K records. To fetch more than 100K records, you need to use the Paging Cookie method in Power Automate. This method can also be used for fetching records below the 100K limit. It retrieves only 5000 records per request, so you need to add requests in a loop and set the Paging Cookie for each iteration. The paging cookie, in XML format, contains the record IDs for the starting and ending records for each iteration, and it changes accordingly for every iteration.</p><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEiq1tFdWcTy6DkzkuW30Ecnc4XUi1lpaJXT-lz-VTGwP7NWtxYb7U6fmn2_PTg3YFaTpcHao9egNLMRKqkRuzqP4MMIcsoo0CiylDg8tTR7Fcwr-aLLaE-IiHQoUljRh5SbaQdiNyTvJ4sm65OcxWJ3SpSgtautHdY68P08bg_eJQtZzK6Nv9czENa5nrM" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_fe35f1855a.png" /></a></p><p></p><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEhK7x4Vrn4vpO55Azak1GIFxeiqxI-bwpRscyWGUUyog0SpExuTMBw1HWyVc2ZnQC1qLHkfQ1XXo5xwjjYFXgSfeVKQaVxyQ0483vwcbjmNfDxR_NEqRByAkLRM6jmfkSKc1-xGqzuJ1G25sRZSZENQ7XnSBZ-Thpv9H8mlDnRQ41Cb7Efuh5oT0Tx96iY" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_9c384a0967.png" /></a></p><p></p><p style="text-align: justify;">We need to initialize several variables, including RecordsCount, PagingCookie, ClearPagingCookie, and EncodingJSON. Within the EncodingJSON object, we create a field named "t" to store the Paging Cookie.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEj6OZ1sGteLDPQT5PkPYbbnWLF4-8VOjvD-gQH5hlfRqZSqxjWvlusKxpHtl2Fti3KteeObId8nKV-FUpEi10d-_KTe4hYmrD-8BiwsIv6HoSDxNuwBfF_7Pp08SAZl18rKEdqnLivIdA5PJ8733Qfdx3lL4k3CVjaEPas9vDldFDvMTonbUB5_3Fm-tEU" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_1940cead64.png" /></a></div><p></p><p style="text-align: justify;">Initiate a "Do Until" action with the condition to check whether the variable PagingCookie is not empty. Use the following expression to evaluate the variable:</p><pre style="text-align: justify;"><code class="language-jsx">empty(variables('PagingCookie'))</code></pre><p style="text-align: justify;">Then inside the "Do Until" loop, add a "List Rows" action for the contact entity with the FetchXML query. In the FetchXML query, include the "page" attribute, which defines the page number. Since the request returns only 5000 records in the first iteration, set the initial page number to 1, and it increments for each subsequent iteration. You can use the below expression to dynamically increment the page number by 1:</p><pre style="text-align: justify;"><code class="language-jsx">add(iterationIndexes('Do_until'),1)</code></pre><p style="text-align: justify;">To set the Paging Cookie, we need to check the iteration index of the "Do Until" loop and then assign the Paging Cookie value stored in the EncodingJSON variable. Use the following expression to set the Paging Cookie:</p><pre style="text-align: left;"><code class="language-jsx">if(equals(iterationIndexes('Do_until'),0),'',concat('paging-cookie=''', substring(first(skip(split(string(xml(setProperty(variables('EncodingJSON'),'t',variables('ClearPagingCookie')))),'&lt;'),1)),2),''''))</code></pre><p style="text-align: left;"></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEhCEgmoec6tb03hiUke2MDywRc3E4jB8sVCOnhxzDr-Ubzep9HUs93gDJGbYZkPRvnjoumUCD1cvTiAFUhlm9TQ8YgIgImvJNq57d3yvUKdQvviTVUsQ0fP0hncCO0VPLNxzlsKa0FaXmV3Oi2BsRsBEZ2WNYhKNxEbus1mPqLA_rPy4--iYgArEnQ30Gg" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_3682909986.png" /></a></p><p></p><p style="text-align: left;">After the "List Rows" action, we need to update the Paging Cookie with the value obtained from the response of the "List Rows" action. Use the following expression to achieve this:</p><pre style="text-align: left;"><code class="language-jsx">if(empty(outputs('List_Contacts')?['body']?['@Microsoft.Dynamics.CRM.fetchxmlpagingcookie']),'',decodeUriComponent(decodeUriComponent(outputs('List_Contacts')?['body']?['@Microsoft.Dynamics.CRM.fetchxmlpagingcookie'])))</code></pre><p style="text-align: left;">To clear the existing Paging Cookie values, use the following expression:</p><pre style="text-align: left;"><code class="language-jsx">if(empty(variables('PagingCookie')),'',replace(substring(variables('PagingCookie'),add(indexOf(variables('PagingCookie'),'pagingcookie="'),14)),'" istracking="False" /&gt;',''))</code></pre><p style="text-align: left;">In this process, we need to retrieve the record count fetched during each iteration. We'll save this count in a variable and increment it by the current count in every iteration.</p><pre style="text-align: left;"><code class="language-jsx">length(outputs('List_Contacts')?['body/value'])</code></pre><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjSiBGbUSqWYiZbasIB5T48ruhJm8G_ZEor_UpIYGtzHwGJgHw_IlPSRb2Uqv9jUYETUajcAQke24e8eqcZnEra6ZfswREUp4X2mSs04FuhC3HsA-eRwcaXMGc9KfEl6eAsm2n47v3VG8dl--3WzoUl1nYiDnIsFI4kLiEnu6wav7_wkl3bPM7vgTGjI9Y" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_8bd12e9f53.png" /></a></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjEyETh39m5MYAIZmOgPLk1U5C6gyoanhs_0AfSvrx6Wh26GG8Em71Z9G4M1Rbv47le04Tiu9w5L3eD2AiYUMBDy07mjn5vir_8uH8waj_UMZVq8H6ZkQZFnlsalkFtzQ1XKlJKvykxuGrKpAuB8jlWDI9vfS7k9PZJ57wVJdfqSTGtLSE69Lab6FQY54I" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_b99be7bcdc.png" /></a></p><p></p><p>Add a "Compose" action and assign the "RecordsCount" variable to it. This will display the final count of records. Save and publish the flow, then run it.</p><p style="text-align: left;">Once the Run is completed, scroll down and expand the "Compose" action to view the actual count of records. In your case, it should display a count potentially exceeding 100K.</p><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjda0EDCr-Cq7wGTWA_LOdrg_WSTcAC9Ga6DIu5eNLl22gqq6Ai7FpNgpzzutdQEapAsR8WpSSRcNGwkB2zWKj3_OfNsWqqp62N1B5yZJInn_kIJCHB6eP1NGX567QTdGv8mLbOAEEvO0AuBFarenHv5_ieWpUv2hgW7c7_bQAXuKbjzVPiHL7TxUguvSM" style="margin-left: 1em; margin-right: 1em;"><img alt="Pagination in Dataverse using Power Automate" src="{{ site.baseurl }}/assets/images/035/img_c8e41d2ab6.png" /></a></p><h2 id="next-link-way">Next Link Way</h2><p style="text-align: justify;"><span>&nbsp;&nbsp; &nbsp;</span>If you're looking to retrieve records exceeding 100,000 without utilizing FetchXML, you can opt for the next link method. This involves leveraging OData queries and incorporating a skip token value, akin to a paging cookie, passed through the OData query parameter. For this approach, it's common practice to initialize two variables: <strong><code>RecordsCount</code></strong> and <strong><code>NextLink</code></strong>.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEiojljHGSKTFLr7tW7QFMtNQqRbvGgqDCS_hHA3VVW_fbYyCVfhky6HTp_Hna1m9mFpUxqF9WsxpTzDasCowizKZHar7OoNTJQgPuYAZ8mDOGdKdGN8JYKyj6w1CTQHD3xCJi-ziVRiS9o4ejJ6HnDWtItUh6VZsDYxdXVPI8qO4LLosBlpeboyXdsE2JY" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="467" data-original-width="712" src="{{ site.baseurl }}/assets/images/035/img_a836344567.png" /></a></div><p></p><p style="text-align: left;">After that, we create a 'Do Until' action to loop through the process until the next link value returns null. As we update the <strong><code>NextLink</code></strong> variable in every iteration, it becomes null when it reaches the last page.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjEnVJlb-eidInHYCUTV909BSQ3vAv0aNHvB64NQJV7WZqWleAHzANgiyzitipCYd-0XNESBsVFYxSgE8raWEU_axrwrYFGP2a62dd3n_JBxShUtPDYyDS3O6zUPHadbyiqgDeHz-7jJR7jh9z2-nhTAf5x7T1zy0MTRwb9ThNsx9CpnzzqQwCjJN3_WzQ" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="228" data-original-width="791" src="{{ site.baseurl }}/assets/images/035/img_d673a26d32.png" /></a></div><p></p><p style="text-align: justify;">Create a "List Rows" action with the table name as "Contacts". Select the "fullname" column. In the "Skip Token" field, add the value of the <strong><code>NextLink</code></strong> variable.</p><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjuL0Jum1r7r854iXDixVEZ-f5jtMf3si3uk7nMG6vYZNRmtHlaJDBDQ9MiVVdP0d3-SnItBE6rMZqC_eMYHq7Qmdoua1h3FTKEr0N-Gmyx4xqz9G8VWew5Npdp4J0SkAdkF5a80XZfhsgLJ-iMb4BSOxC8TeieoCiA5IXDJPwY2oPCn3J2LEjm43VoLLQ" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="500" data-original-width="697" src="{{ site.baseurl }}/assets/images/035/img_2978a566bb.png" /></a></p><p></p><p style="text-align: justify;">Then set the <code>NextLink</code> variable with the value returned from the response. Since we only need the Skip Token parameter, we have to extract it from the <code>$skiptoken=</code> parameter in the next link value.</p><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEjIvYy6kJBZGEisAlR8ZSiAqE49TJEFuuckDhkwbtiymtXyCo4UlwrSVjbCtpjx28CgpgukGfwRxfBaEG9PFzhn1ibDdvaHmGvwSFmKyVcoATHknvi3Dp2g7OkGlqE6r6HHrClUVtUgh8gqY0dMAw_zniKLkG1tB42uy81dg6d4bEd0ImHubpRJblLgWdQ" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="172" data-original-width="693" src="{{ site.baseurl }}/assets/images/035/img_e6315a602d.png" /></a></p><p></p><p style="text-align: left;">You can use the following expression to split the skip token away from the next link.</p><pre style="text-align: justify;"><code class="language-flow">if(not(contains(outputs('List_Contacts')?['body'],'@odata.nextLink')),null,decodeUriComponent(last(array(split(last(array(split(outputs('List_Contacts')?['body/@odata.nextLink'],'&amp;'))),'$skiptoken=')))))</code></pre><p style="text-align: justify;">And increment the count with the current record count from the "List Rows" action to calculate the total count in the end. You can achieve this by using the following expression.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEhrsvgSZK6v-WASkiAwreNJz0G2c2fppUBWEgAu1hZwpozghTuXqFYuRMKFcICoE-IytPAgeT9s6x_ruelf1PJdx6mbXSeCqJaXWd7InhAsvQYFTLhpBOWX8-N95V7DQB6eXNg6ytWuWSH3yxU4Uwjl7dyJL23C-Ng7P-cXpUDx2qJ9_kmNhZ4M_JWvtpE" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="177" data-original-width="691" src="{{ site.baseurl }}/assets/images/035/img_fc12d4b4ba.png" /></a></div><p></p><pre style="text-align: justify;"><code class="language-flow">length(outputs('List_Contacts')?['body/value'])</code></pre><p style="text-align: justify;">Add a "Compose" action outside of the "Do Until" loop and assign the value of the <code>RecordsCount</code> variable to it. This will display the total record count.</p><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEi4g6AOtY6ZbDC6zPdOA-7JCnB21IKm6suIwVJy2cZdqpCJLgTMt6nwyS55EOz15-hSx5ncGfsfKqWPjuEfx4xhGdSXhxoQ5geidv9tKO925ujf4SIZ_iFUsv0aaMF2phD4m6i2YHGexVyzIu6xDx6HGc2JVYhXxtZyZ9U9LxzSoPPQSgOxsPTfJuqt1YA" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="255" data-original-width="712" src="{{ site.baseurl }}/assets/images/035/img_a3f5ba1a0a.png" /></a></div><p></p><p style="text-align: justify;">Run the flow once you've saved and published it. Then open the completed run and expand the "Compose" action that we added last. There, you will be able to see the total record count.</p><p></p><p style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEh9VGStI-jJardckI2dVWaM5sgNcwoviF-uDKqj1tuw8kh3kdQIgA1TOI4ZeT8xLHR7tFCSY77H2-1x5zLingdEail3Khn32f-bHpmFewifv2w-BCFIJbyIHmVw_8VHoSyaQSUdiHwYhnWv-MTNnV66s_vOIkQD7A_zlJ8Sj3r0xzDM7gTnXSoTB46bGsI" style="margin-left: 1em; margin-right: 1em;"><img alt="" data-original-height="448" data-original-width="696" src="{{ site.baseurl }}/assets/images/035/img_7ad7942453.png" /></a></p><p></p><p style="clear: both;"></p><p style="text-align: center;">Have a great day!</p>
