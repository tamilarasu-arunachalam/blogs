During this operation, the resultant record gets created if it doesn't exist or gets updated if it already exist. By default, Dataverse will do the upset operation while using the Web API PATCH. But we must be aware of when do we need upsert and when it is not needed. Upsert can be used in complex data integration processes where we won't know the record already exists or not. At this kind of scenario's upsert would be a great pick. For more details.
