<div style="text-align: justify;">&nbsp; &nbsp; As you all know, Power Automate's action depends on the previous action or trigger. We can even configure the action to run even on the failure of the previous action by using configure run after in settings. But what if we have a scenario of performing multiple actions bundled inside a change set. If any of the action in the set failed, rollback all the completed actions.</div><img border="0" data-original-height="400" data-original-width="700" height="183" src="{{ site.baseurl }}/assets/images/023/img_0437cd6c48.png" style="display: none;" width="320"><p></p><p style="text-align: justify;"><b><i>For Example:</i></b> Creating an Account and Contact in the same set inside a flow. If the creation of contact failed, rollback the creation of account also (actually it got succeeded).</p><p style="text-align: justify;">&nbsp; &nbsp; For this kinda scenario, Dataverse connector had a special action which is <b>'Perform a changeset request'</b>. The changeset request really means of sending multiple requests in a single transaction. We can add other Dataverse actions instead of fields inside this action, it supports three actions such as add, update and delete. The operations inside the changeset request are called atomic, because the whole transaction gets rollbacked if any of the action inside the request is failed.</p><p style="text-align: left;">&nbsp; &nbsp; In this article, I have demonstrated how to use perform changeset request action inside a Power Automate Flow.</p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEj-keCT1XpZRYWV0baPAz0cxotgMXVZcBr3BM8wy_XG4fgDFeWRpMazIsseZh-X9qR6CyTPErUZGKSjd0iUiiJF6E27jjmk8mmmjH9cBbBxiKIL6oYJsSLPV_R8GfBpZN1yanoNr33EXjFPF9SEU_Cj4McPNlBHQ84n0-DvDwKM769WQJevULI8y-uq/s575/seven.png" style="margin-left: 1em; margin-right: 1em;"><img border="0" data-original-height="217" data-original-width="575" height="151" src="{{ site.baseurl }}/assets/images/023/img_514b42837f.png" width="400"></a></div><p style="text-align: left;"></p><ul style="text-align: left;"><li>&nbsp;To create a flow, go to make.powerapps.com&nbsp;<span style="text-align: left;">🡲&nbsp;Solutions&nbsp;</span><span style="text-align: left;">🡲&nbsp;+New&nbsp;</span>🡲 Automation 🡲 Cloud Flow&nbsp;🡲 Instant.</li></ul><p></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEi15ETc4wIftwi4UdAOUnuBLB6J7CiYu_pILuLIpMrGbRdL5pStXCVLsDSUosHjMxj-ZmGV3hi5IW_53KicnFQ_PxHTR68zXV8mQIkgBeOARNirg9PoOIipHaZZlRAggugf8SqUUFI5O_5m-O-HVt4FptKj7K3ZhktAfAbnS4SjZtdmEL8Z82iYUFag/s1600/two.PNG" style="margin-left: 1em; margin-right: 1em;"><img border="0" data-original-height="646" data-original-width="1014" height="255" src="{{ site.baseurl }}/assets/images/023/img_3340ec6d7e.png" width="400"></a></div><div class="separator" style="clear: both; text-align: left;"><p></p><ul style="text-align: left;"><li>A prompt opens for selecting the name of the flow and trigger.
